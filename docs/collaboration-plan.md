# Collaboration Plan

## Positioning

Use this wording:

> Flyto2 Health Twin is an open, non-diagnostic personal physiological response
> modeling prototype. It uses wearable data, lifestyle logs, and optional
> health metrics to predict recovery signals and identify missing variables
> through transparent error analysis.

Avoid:

- Longevity cure
- Telomere repair
- Stem-cell injection
- Whole-human simulation
- Medical diagnosis or treatment

## Who To Approach

- Biomedical engineering labs
- Medical informatics programs
- Preventive medicine
- Family medicine
- Rehabilitation and sports medicine
- Wearable data researchers
- Device vendors
- Student research groups

## First Ask

Ask for feedback, not equipment:

```text
I built a local open-source prototype that predicts next-day HRV, resting heart
rate, fatigue, and sleep quality from privacy-preserving daily aggregates. The
most important output is not accuracy claims; it is transparent error analysis
that shows which variables may be missing.

Would you be willing to review the model framing and suggest what biomarker or
device data would make this more scientifically useful?
```

## Evidence Before Equipment

Before asking for devices, prepare:

- 30 to 90 days of complete daily aggregate logs.
- Public dashboard export.
- Prediction error history.
- Missing-variable report.
- Clear privacy policy.
- GitHub repository with synthetic data and reproducible CLI.
