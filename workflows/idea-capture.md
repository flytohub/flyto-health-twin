# Idea Capture Workflow

Use this when turning a rough idea into a tracked Flyto2 task.

1. Classify the idea as importer, model, public export, registry/gate,
   simulation, dashboard, or research documentation.
2. State the non-medical purpose and whether any private health data is needed.
3. Identify input schema, public allowlist, benchmark, device gate, and UI impact.
4. Use Flyto2 Indexer to locate owning symbols and affected references.
5. Move accepted work into `ROADMAP.md`, `tasks.md`, or a dated handoff.
