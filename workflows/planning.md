# Planning Workflow

Use this before implementation work.

1. Read the project boundary, data model, privacy policy, architecture, state,
   decisions, and recent handoffs.
2. Run Flyto2 Indexer context and impact analysis on affected Go and web symbols.
3. Define schema compatibility, chronology, privacy, benchmark, and rollback
   criteria before implementation.
4. Identify required generated references, tests, CLI smoke commands, public
   export checks, docs, and deployment surfaces.
5. For frontend work, define mobile/desktop, loading/error, accessibility, and
   metadata acceptance checks.
